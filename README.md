# 🎯 ML Model Accuracy Evaluator

Скрипт на чистом Python для оценки точности (**accuracy**) классификатора — принимает предсказания модели и истинные метки, считает долю правильных ответов и корректно обрабатывает краевые случаи вместо падения с ошибкой.

## Задача

После обучения классификатора нужно измерить, насколько хорошо он угадывает. Базовая метрика для этого — **accuracy**: доля совпадений между тем, что предсказала модель, и тем, что есть на самом деле.

Цель проекта — не просто посчитать процент, а написать **надёжную** функцию, которая не ломается на «грязных» входных данных: пустых списках, списках разной длины и т.п.

## Что считает скрипт

**Accuracy** — это доля верных предсказаний:

```
accuracy = (число совпавших пар) / (общее число пар)
```

Пример: если из 5 предсказаний 4 совпали с реальными метками, accuracy = 4 / 5 = **0.8 (80%)**.

## Как устроено

1. На вход подаются два списка одинаковой длины: `predicted` (предсказания модели) и `actual` (истинные метки).
2. Через `zip()` оба списка проходятся **параллельно**, пара за парой.
3. Счётчик считает, в скольких парах значения совпали.
4. Число совпадений делится на общее количество пар.
5. Перед расчётом работает **валидация входных данных** — см. ниже.

```python
def calculate_accuracy(predicted, actual):
    # Валидация: списки не должны быть пустыми
    if len(predicted) == 0 or len(actual) == 0:
        raise ValueError("Списки не могут быть пустыми")

    # Валидация: длины должны совпадать
    if len(predicted) != len(actual):
        raise ValueError("Длины списков должны совпадать")

    correct = 0
    for pred, true in zip(predicted, actual):
        if pred == true:
            correct += 1

    return correct / len(actual)
```

## Обработка ошибок (edge cases)

Главная идея проекта — **не падать с исключением, а явно обрабатывать проблемные случаи**:

| Случай | Поведение |
|--------|-----------|
| Пустые списки | Возбуждается `ValueError` с понятным сообщением |
| Разная длина списков | Возбуждается `ValueError` — accuracy в таком случае посчитать нельзя |
| Корректные данные | Возвращается значение accuracy от 0.0 до 1.0 |

## Тестовые сценарии

Скрипт проверяется на 5 сценариях, включая краевые:

1. **100% точность** — все предсказания совпадают с метками → `1.0`
2. **0% точность** — списки одинаковой длины, но все значения разные → `0.0`
3. **Частичное совпадение** — часть предсказаний верна → дробное значение (например, `0.6`)
4. **Пустые списки** → ожидаемая ошибка `ValueError`
5. **Разная длина списков** → ожидаемая ошибка `ValueError`

## Пример использования

```python
predicted = ["spam", "not_spam", "spam", "spam", "not_spam"]
actual    = ["spam", "not_spam", "not_spam", "spam", "not_spam"]

print(calculate_accuracy(predicted, actual))  # 0.8
```

## Инструменты

- **Python** (чистый, без внешних библиотек)
- `zip()` — параллельный проход по двум спискам
- `try / except` и `raise` — обработка ошибок
- Ручное тестирование на наборе сценариев
